# AGENTS.md

## 언어

설명, 문서, 사용자 UI는 한국어를 기본으로 한다. 코드 식별자와 표준 기술 용어는 영어를 사용할 수 있다.

## 제품 기준

문서 우선순위는 다음과 같다.

1. `docs/specs/2026-07-19-neon-breach-v2-product-contract.md`
2. `docs/specs/2026-07-19-neon-breach-v2-story.md`
3. `docs/plans/2026-07-19-neon-breach-v2-execution-plan.md`
4. V1 설계·계획 문서

V1 문서는 역사 기준이다. V2와 충돌하는 10분 런, `처치`, 아군 소멸, 단순 재접속 문구를 복원하지 않는다.

## 현재 범위

- 먼저 CHAPTER 1 수직 슬라이스만 완성한다.
- 새 일반 적, 완전히 새로운 보스, 분기 선택, 허브는 CH1 사람 검증 전 추가하지 않는다.
- 기존 전투 엔진은 재사용한다. 렌더러·입력·투사체를 이유 없이 재작성하지 않는다.
- `main.js`에 새 콘텐츠와 DOM 문자열을 계속 쌓지 않는다.

## 아키텍처 불변조건

- `RunState`와 `ProfileState`를 분리한다.
- 실행 중 상태는 `localStorage`에 저장하지 않는다. 완료 기록, 대사 `seen`, 기록 보관소, 통신 설정만 저장한다.
- 페이지 새로고침으로 재시작하지 않는다. 모든 런 모듈을 명시적으로 초기화한다.
- 챕터 콘텐츠에는 DOM 조작이나 실행 콜백을 넣지 않는다. ID, 수치, 문구와 트리거만 둔다.
- 자동 공격은 `차단`, SPACE 해킹은 공공 구조 프로토콜 `복구`로 표현한다.
- 여섯 번째 드론은 삭제하지 않고 현장 파견 결과를 반환한다.

## 자율 진행 규칙

제품 계약의 휴먼 게이트 사이에서는 다음 작업을 묻지 않고 진행한다.

- 내부 모듈명과 파일 분리
- 테스트·lint·CI 추가 및 실패 수정
- 계약 범위 안의 버그 수정과 P0/P1 리뷰 반영
- 브라우저 smoke, 전체 런 QA, 빌드 검증
- 브랜치 내부의 단계별 커밋과 문서 갱신(사용자가 실행 계획 수행을 요청한 경우)

다음 경우에는 멈추고 사람 결정을 받는다.

- 정사, 제품명, 출시 형태를 바꾸는 결정
- 플레이 감각과 서사 이해도를 승인하는 CH1 수직 슬라이스 검수
- 기존 저장 데이터를 버리는 마이그레이션
- 외부 에셋·과금·계정·개인정보·새 플랫폼 추가
- `main` 반영 또는 공개 배포

## 검증

검증 기반이 추가되기 전 최소 명령:

```bash
npm run build
```

Phase 1 이후 필수 명령:

```bash
npm run lint
npm test
npm run build
```

Phase 7에서 브라우저 러너가 추가된 뒤에는 `npm run check:full`까지 통과한다.

CH1 통합 이후에는 8분 전체 런, 패배 후 다시 시도, 승리 후 진행도 유지, 콘솔 오류 0을 브라우저에서 확인한다. 빌드만 통과한 상태를 플레이 완료로 보고하지 않는다.

## Git과 배포

현재 `main` 푸시는 GitHub Pages 배포를 시작한다. 명시적 출시 승인 없이 `main`에 push 또는 merge하지 않는다. 사용자 파일과 무관한 `.DS_Store`, IDE 설정, 기존 미추적 파일을 작업 범위에 섞지 않는다.
